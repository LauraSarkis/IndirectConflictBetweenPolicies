
Algumas Instruções.

Esta pasta foi disponibilizada para que fosse possivel verificar como ocorrem a verificação das políticas nos arquivos owl.

Os arquivos disponibilizados são referentes a cada um dos experimentos testados e que constam na tese.

O único arquivo csv disponibilizado é o arquivo que a IES disponibilizou e preserva a estrutura fornecida. 
    Para testá-lo deve-se utilizar a ferramenta CSVOWL - Conversor.jar, que vai convertê-lo para o formato owl.
    Os passos na ferramenta são: 
          Browse- para escolher o arquivo. 
          Load - para verificar se está carregado corretamente (este item tem que ser cumprido para passar para os próximos passos)
          Create OWL - tendo em vista que este arquivo tem todos os dados suficientes do teste.
          Vai ser solicitado um nome para o arquivo se salvo.

Todos os arquivos owl podem ser utilizados na ferramenta Conflict Detector, cujos passos para observar a detecção de conflitos compreende:

		FILE-> OPENOWL -> ESCOLHA DO ARQUIVO OWL (GERALMENTE VAI BUSCAR NA PASTA DOCUMENTOS).
                
		O ARQUIVO APARECE CARREGADO NA TELA.

		PARA VERIFICAR OS CONFLITOS NA FERRAMENTA ESCOLHER: CONFLICTS->CHECK IN POLICIES INSIDE FILES

                NA NOVA TELA QUE APARECER CLICAR EM CHECK CONFLICTS
                
		O RESULTADO APARECERÁ E A BARRA DE ROLAGEM DO LADO DIREITO, PERMITE QUE POSSA SER TODO VISUALIZADO, ELE PODE SER EXPORTADO PARA TXT OU SIMPLESMENTE, CASO QUEIRA VOLTAR PARA A TELA PRINCIPAL TECLAR EXIT. E PARA SAIR DO PROGRAMA EXIT NOVAMENTE.
